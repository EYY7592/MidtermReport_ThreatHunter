§2 · 誠實弱點評估（不美化）

⚠️ 這一節很重要：有弱點不代表不好，但要清楚知道風險在哪裡。

❌ 弱點 1：AMD Hardware 整合未實測

說明：我們聲稱「在 AMD MI300X 上用 vLLM 跑 Llama-70B」，但目前開發環境用的是 OpenRouter API。

風險：AMD Hackathon 技術應用分數可能低，因為評審強調「深度整合而非 API 包裝」。

解決方案：申請 AMD Developer Cloud 帳號，實際部署一次，取得效能截圖作為證明。

時間：需 4-8 小時配置。

❌ 弱點 2：攻擊連鎖分析精確度無驗證

說明：Analyst Agent 的 SSRF→Redis→RCE 連鎖推理是 LLM 輸出，可能有幻覺。

風險：評審可能問「這個連鎖分析準確率是多少？」我們沒有數字。

解決方案：在 DVWA 或 VulnHub 上跑 5-10 個已知攻擊鏈，計算 Precision/Recall。

時間：需 6-12 小時測試。

⚠️ 弱點 3：ColMAD 加入的辯論時間和 Token 成本

說明：CrewAI Hierarchical 比 Sequential 的 Token 消耗更高（Manager 有額外推理開銷）。[6]

來源：dev.to — CrewAI Token 優化指南。

緩解：三方辯論一致時跳過 Phase 2（快捷路徑設計），省 6 次 LLM 呼叫。

評估：設計有考慮到，但沒有實際測量節省多少。

⚠️ 弱點 4：EPSS 和 OTX 的可靠性限制

說明：EPSS 對硬體漏洞（如 BIOS/韌體）的預測不準確。OTX 是社區回報，品質不一。

來源：arXiv EPSS 效能研究（「某些硬體漏洞維持低 EPSS 但仍被積極利用」）。

緩解：Intel Fusion Agent 對 OTX 設低權重（0.05），並動態調整 EPSS 權重。

⚠️ 弱點 5：展示材料未完成

說明：FastAPI Live URL 部署 + Pitch 影片 + Demo 影片 尚未完成。

UI 架構：已採用 FastAPI + SSE + Vanilla HTML，支援即時 Agent 監控串流（已優於 Streamlit）。

風險：Presentation 評分可能低。評審要求 Live URL 和 5 分鐘影片。

優先度：這是最高優先的待辦事項之一。

⚠️ 弱點 6：Hackathon 期間無生產記錄

說明：ThreatHunter 是 Hackathon 專案，沒有生產環境使用記錄。

緩解：用 DVWA 測試取得可量化的 Precision/Recall 數據作為替代。

✅ 最終架構裁定：方向正確，執行有缺口

架構是否是同類中最好的？ 在「開源 LLM 安全分析 Agent」這個細分域內， ThreatHunter 的設計組合（MacNet DAG + LLM Discussion + ColMAD + 六維融合） 在公開研究中找不到完全相同的組合，具備原創性。

核心結論：架構設計水準足夠打進前三名，但展示執行不到位就失分。

§7b · UI 架構說明（為何不用 Streamlit）

❌ Streamlit 的根本限制

限制具體問題對 Demo 的影響
無真實即時串流 Streamlit 靠 polling 或 st.empty() 更新，不是 WebSocket/SSE
Agent 執行中無法看到每個步驟的即時輸出		 評審看不到「六個 Agent 依序工作」的視覺化過程
阻塞式執行 Streamlit ScriptRunner 是單執行緒，長任務會凍結整個 UI 掃描 60-120 秒期間 UI 無回應，體驗差
多用戶共享狀態 Session state 在多用戶場景容易污染 Demo 時多人同時試用會互相影響

✅ 現行架構：FastAPI + SSE + Vanilla HTML/JS

技術堆疊

fastapiREST API 後端 (POST /api/scan)
uvicornASGI 伺服器，支援非同步
SSE（Server-Sent Events）GET /api/stream/{scan_id}
Agent 每完成一步即時推送事件
Vanilla HTML/CSS/JS靜態前端，無框架依賴
threadingPipeline 在背景執行緒跑，UI 不阻塞

SSE 事件類型（即時監控訊號）

connected連線成功，確認 scan_id
agent_start某個 Agent 開始執行
agent_logAgent 執行中的即時日誌
agent_doneAgent 完成，含 status + detail
done整個 Pipeline 完成，返回完整報告
pipeline_error錯誤訊息，前端立即顯示
比 Streamlit 好在哪裡： 用戶點擊「開始掃描」後，瀏覽器立即透過 SSE 長連線接收事件流。 Security Guard「開始提取」→ Intel Fusion「查詢六維情報」→ Analyst「分析連鎖」→ ColMAD「三方辯論中」—— 每一步都實時出現在畫面上，評審可以親眼看到七個 Agent 協作工作的全過程。

啟動指令（開發環境）

uv run uvicorn ui.server:app --host 0.0.0.0 --port 1000 --reload

啟動指令（生產環境）

uv run python ui/server.py

部署目標：Railway / Render / AMD Developer Cloud（FastAPI 部署無需特殊設定，比 Streamlit Cloud 更彈性）
